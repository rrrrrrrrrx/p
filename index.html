Overview

What it is

Prophet runs outside the game process (typical “external” setup): driver/memory access to read game state and an on-screen menu. It is not a kernel driver and does not claim to be invisible to anti-cheat.

◇
External architecture
Runs as its own program with memory access to Roblox—not injected code inside the game client. That model trades some capabilities for separation from the game’s address space; it is still subject to detection like any third-party tool.
↳ User-mode external
◇
Modules
Toggles grouped by combat, visuals, and misc (aim assist, ESP, movement helpers, overlay, config, etc.). The count changes with updates—think on the order of dozens of features, not a fixed marketing number.
↳ Combat / visuals / misc
◇
Updates
When a new build is published, the version and download URL live in update.json. A loader can poll that file so clients don’t rely on manual links every time.
↳ Manifest-driven
◇
Overlay & config
In-game style menu for settings, plus saving and loading configs. Performance impact depends on your machine and what you enable.
↳ Menu + profiles
